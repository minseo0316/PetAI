Pet.AI 프로젝트 개요

Pet.AI는 사용자가 제공한 반려동물의 정보(텍스트, 이미지)를 바탕으로 Google의 Gemini AI 모델을 활용하여 건강 상태에 대한 예비 분석과 조언을 제공하는 웹 애플리케이션입니다. 사용자 인증 시스템을 통해 개인화된 분석 기록을 관리하고, AI 챗봇을 통한 추가적인 건강 관리 기능을 제공하여 실용성을 높였습니다.

핵심 기능

AI 종합 분석 (RAG 패턴 적용)

단순히 AI에게 질문하는 것을 넘어, AI의 '환각 현상'을 보완하기 위해 **검색 증강 생성(RAG)** 패턴을 적용했습니다. AI가 사진을 1차 분석한 결과를 바탕으로 자체 DB에서 관련 질병 정보를 검색하고, 이 정보를 참고하여 최종 답변을 생성함으로써 분석의 정확성과 일관성을 높였습니다.

예시 프롬프트:

SYSTEM

당신은 전문 고양이 수의사 AI 조수입니다.

CONTEXT (DB 검색 결과)
[{'disease_name': '알레르기성 피부염 (의심)', 'advice': '...'}]
USER (보호자 관찰 내용)

최근 그루밍을 너무 과하게 하고, 특정 부위를 계속 핥아요.

TASK (임무)

위의 [CONTEXT]를 바탕으로 [USER]의 질문에 답변하세요. (규칙 및 출력 형식)...

사용자 인증 및 개인화된 기록 관리

회원가입 및 로그인 기능을 구현하여 사용자별로 과거 분석 기록을 안전하게 저장하고 언제든지 다시 확인할 수 있습니다. 이를 통해 반려동물의 건강 상태 변화를 시간순으로 추적할 수 있습니다.

AI 영양사 챗봇

사용자와의 대화를 통해 반려동물의 정보를 수집하고, 이를 바탕으로 맞춤형 다이어트 계획을 제안하는 목적 지향적 AI 챗봇 기능을 제공합니다. 모든 대화 내용은 DB에 저장되어 대화가 이어질 수 있습니다.

동적이고 직관적인 UI/UX

AI가 분석한 결과의 위험도(안전, 주의, 경고, 상담)에 따라 배경색이 동적으로 변경되어 사용자가 결과를 한눈에 파악할 수 있도록 설계했습니다. 또한 모든 페이지를 모바일 환경에서도 편하게 사용할 수 있도록 반응형으로 구현했습니다.

나의 역할 및 기여

이 프로젝트에서 저는 풀스택(Full-Stack) 개발자 및 DevOps 엔지니어 역할을 수행하며 프로젝트의 기획부터 설계, 개발, 배포의 전 과정을 주도했습니다.

백엔드 시스템 설계 및 개발

  • Flask를 사용하여 사용자 인증, 분석, 기록 관리, 챗봇 등 모든 비즈니스 로직을 처리하는 RESTful API를 설계하고 개발했습니다.
  • AI 답변의 신뢰도를 높이기 위해, 이미지 분석 결과를 바탕으로 자체 DB를 검색하여 AI 프롬프트에 동적으로 주입하는 **RAG(검색 증강 생성) 파이프라인**을 직접 구현했습니다.

데이터베이스 설계 및 관리

  • 사용자(users), 질병(diseases), 분석 기록(analysis_history), 챗봇 대화(chat_history) 테이블을 설계하고, 관계형 데이터베이스 스키마를 구축했습니다.
  • `ON DELETE CASCADE` 옵션을 적용하여, 사용자 탈퇴 시 관련 데이터가 모두 자동으로 삭제되도록 데이터 무결성을 확보했습니다.

DevOps 및 배포 자동화

  • Render 플랫폼에 성공적으로 프로젝트를 배포하고, GitHub 푸시 시 자동으로 재배포되는 CI/CD 파이프라인을 구축했습니다.
  • 로컬(SQLite)과 배포(PostgreSQL) 환경을 모두 지원하도록 코드를 설계하고, 환경 변수를 통해 DB 정보와 API 키를 안전하게 관리했습니다.

회고 (Retrospective)

배운 점 (What I Learned)

  • RAG 패턴의 강력함: 단순 API 호출을 넘어, 자체 데이터를 결합하는 RAG 패턴이 AI의 환각을 줄이고 서비스의 신뢰도를 극적으로 향상시킨다는 것을 직접 경험했습니다.
  • 에러 핸들링의 중요성: 간헐적으로 발생하는 500 서버 오류를 해결하는 과정에서, 명확한 오류 로깅과 안정적인 모델 선택, 타임아웃 설정 등 실전적인 디버깅 및 서버 안정화 기술을 체득했습니다.
  • Full-Stack 개발의 시야: 프론트엔드부터 백엔드, 데이터베이스, 배포까지 전 과정을 직접 경험하며, 각 기술 요소가 어떻게 유기적으로 연결되어 하나의 서비스를 완성하는지 깊이 이해하게 되었습니다.

개선점 (What Could Be Improved)

  • 이미지 영구 저장: 현재는 서버 재시작 시 업로드된 이미지가 사라집니다. Amazon S3와 같은 외부 클라우드 스토리지를 도입하여 이미지를 영구적으로 보존하는 기능이 필요합니다.
  • 코드 모듈화: `app.py` 파일이 점차 비대해지고 있습니다. 데이터베이스 관련 로직, AI 호출 로직 등을 별도의 파일로 분리하여 코드의 가독성과 유지보수성을 높일 필요가 있습니다.
  • 비동기 처리 도입: AI 분석은 시간이 오래 걸릴 수 있으므로, Celery나 Redis 같은 큐 시스템을 도입하여 분석 요청을 비동기적으로 처리하면 사용자 경험을 크게 향상시킬 수 있습니다.
