Pet.AI 프로젝트 개요

Pet.AI는 사용자가 제공한 반려동물의 정보(텍스트, 이미지, 수치 데이터)를 바탕으로 Google의 Gemini AI 모델을 활용하여 건강 상태에 대한 조언을 제공하는 웹 애플리케이션입니다.

프로젝트 아키텍처 및 설계

프로젝트는 다음과 같은 아키텍처로 구성되어 있습니다.

[Render (PaaS)]

[AI/DB 모듈]

  • requirements.txt (의존성)

핵심 기능

다중 입력 기반 분석

반려동물의 사진, 텍스트 증상, 나이/체중, 특정 이상 행동 등 복합적인 정보를 제공하여 보다 정확한 분석을 요청할 수 있습니다.

AI 종합 소견 제공

Gemini 모델을 활용하여 입력된 모든 정보를 종합적으로 판단하고, '핵심 요약', '상세 설명', '권장 조치' 형식으로 구조화된 AI 소견을 생성합니다.

동적 프롬프트 엔지니어링

사용자의 입력(사진 유무, 증상 텍스트 유무)에 따라 Gemini에 전달되는 프롬프트의 내용과 임무(mission)가 동적으로 변경되어, 각 상황에 가장 적합한 답변을 생성하도록 설계되었습니다.

예시 프롬프트:

당신은 전문 고양이 수의사 AI 조수입니다.

[사진 분석 결과 라벨]
피부 발진

[사진 분석과 관련된 수의학 지식 (DB 검색 결과)]
[{'disease_name': '알레르기성 피부염 (의심)', 'advice': '사진과 증상으로 볼 때...'}]

[보호자 관찰 내용]
최근 그루밍을 너무 과하게 하고, 특정 부위를 계속 핥아요.

부가 분석 기능

선택된 이상 행동의 원인과 보호자 코칭을 제공하고, 나이와 체중을 기반으로 비만도를 체크하여 사용자에게 추가적인 건강 정보를 제공합니다.

나의 역할 및 기여

이 프로젝트에서 저는 풀스택(Full-Stack) 개발자 및 DevOps 엔지니어 역할을 수행하며 프로젝트의 기획부터 설계, 개발, 배포의 전 과정을 주도했습니다.

백엔드 시스템 설계 및 개발

  • Flask를 사용하여 안정적인 웹 서버를 구축하고, 사용자의 다양한 입력을 처리하는 /analyze API 엔드포인트를 설계했습니다.
  • Google Gemini API를 연동하고, 답변의 품질과 안전성을 높이기 위해 상세한 규칙과 출력 형식을 포함한 동적 프롬프트 엔지니어링을 수행했습니다.

프론트엔드 및 UI/UX 개선

  • 핵심 기여: Gemini가 생성한 마크다운 텍스트가 웹에서 깨지지 않고 제목, 목록 등이 포함된 서식 있는 텍스트로 렌더링되도록 markdown 라이브러리와 | safe 필터를 적용하여 가독성을 크게 향상시켰습니다.

DevOps 및 배포 자동화

  • Git을 사용한 버전 관리 시스템을 구축하고, 충돌(Conflict) 해결, 병합(Merge) 등 협업의 기반을 마련했습니다.
  • Render 플랫폼에 성공적으로 프로젝트를 배포하고, CI/CD 파이프라인을 구축했습니다. (Procfile, Gunicorn, requirements.txt, DB 자동 생성, 환경 변수 관리 포함)

회고 (Retrospective)

배운 점 (What I Learned)

  • 배포는 개발의 연장선: 로컬에서는 발생하지 않던 실전적인 DevOps 문제들을 해결하며 안정적인 서버 운영의 중요성을 체감했습니다.
  • Git 버전 관리의 필요성: 충돌과 에러를 직접 해결하며 원격/로컬 저장소 동기화 원리와 협업의 기반을 몸소 깨닫게 되었습니다.
  • 생성형 AI 연동의 핵심: 단순 API 호출을 넘어, 정교한 프롬프트 엔지니어링이 AI의 성능을 극적으로 좌우한다는 것을 경험했습니다.

개선점 (What Could Be Improved)

  • 실제 이미지 분석 모델 도입: 현재의 가짜 분석을 넘어, Google Vision API나 커스텀 모델을 도입하여 서비스의 정확성을 높일 필요가 있습니다.
  • 사용자 인증 및 분석 기록 관리: 회원가입 기능을 추가하여 과거 분석 기록을 추적할 수 있다면 더욱 실용적인 서비스가 될 것입니다.
